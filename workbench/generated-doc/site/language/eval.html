    Evaluation

    Boolean Evaluation

    Almost any expression can be used as an argument for boolean constructs such as filters, conditionals or logical operators. Due to the fact that an expression evaluates to a sequence of nodes, it is necessary to precisely define its evaluation to an boolean value.

    The boolean function test maps sequence \(S\) to a boolean value:

    1. )   if \(S\) is a sequence with more than one element, the boolean value is true
    2. )   if \(S\) is a singleton value \(e\) and
      1. )   \(e\) of type boolean, then the boolean value is \(e\)
      2. )   otherwise the boolean value is true
    3. )   if \(S\) is an empty sequence, the boolean value is false

    Processing Table

    The evaluations of essential, compound expressions whose evaluations could not be immediately apparent, are the focus of this section.

    The semantic function eval maps an expression X and a context node \(o\), which is initially the input node, to a sequence of nodes, and is partially defined in the following table.

    language element expression pattern evaluation
    context node: \(o\)
    shortcut!: eval(X) = eval(X, \(o\))    		 remark
                                                                                          
    self _ return \(o\)
    scope expression X, Y eval(X)
    return eval(Y)    		 X,Y,Z \(=\) ((X,Y),Z)
    conditional if (B) X Y if (test(eval(B))) then return eval(X)
    else return eval(Y)
    path X.Y let \(R\) be the empty sequence
    for every \(x\) in eval(X) do
    \(\;\;\;\)for every \(y\) in eval(Y, \(x\)) do
    \(\;\;\;\;\;\;\)include \(y\) in \(R\)
    return \(R\)    		 X.Y.Z \(=\) ((X.Y).Z)
    filter X?(Y) let \(R\) be the empty sequence
    for every \(x\) in eval(X) do
    \(\;\;\;\)if (test(eval(Y, \(x\)))) then include \(x\) in \(R\)
    return \(R\)
    construction (object) { p : X, ... } return { p : eval(X), ... } p is an identifier or string
    ... dynamic property name { (X) : Y, ... } return { eval(X) : eval(Y), ... } X must evaluate to a string
    ... embedding { X, ... } return { properties(eval(X)), ... } X must evaluate to an object. properties(\(x\)) 'embeds' the properties of \(x\) into the enclosing object
