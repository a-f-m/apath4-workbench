    Description by Grammar

    apath is an expression language, so the start rule is Expression. For instance, person.age == 18 checks the equality of a value of a path with a literal value. Because a path with steps is the most commonly used and central construct, the grammar here will start at this level. The grammar will be described top-down1.

    Icon references the related cheat-sheet part. See also basic definitions.

    Paths

    Paths consist of 'dot'-separated composite steps. Every step yields a sequence of nodes that are passed one by one to the subsequent step. This passed node is called context node. The solution of a path is the yielded sequence of the last step. If one step yields an empty sequence, no further steps are performed, and the solution of the whole path is the empty sequence. See evaluation for details.

    ... '$' ...

    Composite Steps

    A composite step is solely a basic step or its filtering or subscription (if it represents an array).

    Filter & Subscript

    By means of a filter, sequences retrieved by the base step, can be restricted to sub-sequences of nodes that fulfill the stated expression.

    Subscripts are reserved for arrays, and the stated expression must evaluate to a number.

    Basic Steps

    A basic step is a step function call, a property value selection, all children, a parenthesized expression, or the context node itself. Furthermore, objects and arrays can be created.

    Step Function Call

    Step function calls take expressions as arguments and yield sequences defined by the function. They are prominently used for extending apath, see Step Functions for details.

    Properties

    Property values of an object can be selected by an identifier that must not be a reserved word. If the property name is not a regular identifier (e.g. #1), a so-called name literal can be used. If a regex is given, all properties with names conforming to it will be taken.

    Construction

    The construction of objects and arrays conforms to JSON/js syntax with special enhancements.

    Objects

    As usual, an object is built with a list of property assignments. The property value (right-hand side of an assignment) is an expression whose value must be a singleton or empty.2 If it is empty, no assignment is performed. The property name can be at least an identifier or string -- we adopt the relaxed definition from js instead of JSON.

    If a property name shall be dynamically created, a parenthesized expression has to be used. In this case, its evaluation must be a string.

    So-called embeddings can be realized by omitting the property name. In this case, the value of the expression can be a sequence with more than one element. The properties of every node in this sequence are embedded in the constructed object, possibly overwriting previous properties.3

    Arrays

    The array constructor takes a list of expression values as consecutive array elements. If one value is not a singleton, all its elements will be embedded.

    Expressions

    Composite Expressions

    Expressions can be sequenced, delimited by a comma. In this case, its value is the value of the last expression. It is mostly used for side effects, e.g., logging.

    The conditional expression consists of the condition-part, then-part, and optional alternative4.

    Basic Expressions

    Basic expressions, starting from LogicalORExpression, follow the usual logical and relational expression syntax of programming languages . apath mostly reuses javascript syntax, adopted from the js-PEG-grammar with slight changes.

    Logical expressions use keywords and/or/not (instead of &&, ||, !) and equality/relational expressions (==, !=, <=, >=, <, >) are defined as usual, see compact grammar for details. Also, as usual, the atomic parts of an expression are primary expressions, and they now contain paths:

    Compact EBNF Grammar

    Main EBNF rules in PEG (Parsing Expression Grammars) style. 

    //-
//------------------------ Composite Steps ------------------------
//-
Path =
    CompositeStep ('.' CompositeStep)*
CompositeStep =
    Filter / Subscript / BasicStep
Filter =
    BasicStep '?' '(' Expression ')'
Subscript =
    BasicStep '[' Expression ']'
//-
//- ------------------------ Basic Steps ------------------------
//-
BasicStep = 
    StepFunctionCall / Property / Children 
    / '(' Expression ')' 
    / Self
    / Construction
Property =
    Identifier
    / NameLiteral
    / RegularExpressionLiteral
Children =
    '*'
Self =
    ('self' / '_')
StepFunctionCall =
    Identifier '(' ArgumentList? ')'
ArgumentList =
    OrdinaryExpression (',' OrdinaryExpression)*
//-
//- ------------------------ Basic Steps / Construction ------------------------
//-
Construction =
    ObjectConstruction
    / ArrayConstruction
ObjectConstruction =
    "{" PropertyAssignmentList? "}" 
PropertyAssignmentList =
    PropertyAssignment ("," PropertyAssignment)* 
PropertyAssignment =
    (PropertyName ":")? OrdinaryExpression 
PropertyName =
    Identifier
  / StringLiteral
  / '(' Expression ')'
ArrayConstruction =
    '[' ElementList? ']'
ElementList =
    OrdinaryExpression (',' OrdinaryExpression)*
//-
//------------------------ Expressions ------------------------
//-
Expression = SequencedExpression
SequencedExpression =
    OrdinaryExpression (',' OrdinaryExpression)*
OrdinaryExpression =
    ConditionalExpression
ConditionalExpression =
    'if' '(' Expression ')' OrdinaryExpression OrdinaryExpression?
    / LogicalORExpression
LogicalORExpression =
    LogicalANDExpression ('or' LogicalANDExpression)*
LogicalANDExpression =
    EqualityExpression ('and' EqualityExpression)*
EqualityExpression =
    RelationalExpression (EqualityOperator RelationalExpression)*
EqualityOperator =
    "=="
    / "!="
RelationalExpression =
    AdditiveExpression (RelationalOperator AdditiveExpression)*
RelationalOperator =
    "<="
  / ">="
  / "<"
  / ">"
AdditiveExpression =
    MultiplicativeExpression (AdditiveOperator MultiplicativeExpression)*
AdditiveOperator = 
    "+"
  / "-"
MultiplicativeExpression =
    ExponentiationExpression (MultiplicativeOperator ExponentiationExpression)*
MultiplicativeOperator =
    "*"
  / "/"
  / "%"
ExponentiationExpression =
    UnaryExpression ("**" ExponentiationExpression)*
UnaryExpression =
    PrimaryExpression
  / UnaryOperator UnaryExpression 
UnaryOperator =
    "+"
  / "-"
  / "not"
PrimaryExpression =
    Literal
  / Path

    1. non-Terminals are hyper-linked if defined here. ↩

    2. Other path languages allow sequences with more than one element, which are automatically resolved to an array. We omit this functionality for clarity. If this behavior is intended, the right-hand side can simply use the array constructor [...]. ↩

    3. By this an extension of objects can be easily done, e. g., the expression pattern {_, ...} extends the context node (self) by subsequent assignments. Rem.: Future versions of apath will support paths at the left hand side of assignments such that deep assignments can be easily defined. ↩

    4. As in some other functional languages conditionals are without the else keyword. For complex parts, parsing unambiguousness, and better readability you can use parenthesized expressions, e. g., if (A) (a, b) (c + 1). Only very simple ones could skip them. ↩
